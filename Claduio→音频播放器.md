## 项目概述

Claduio 是私人 AI 电台 DJ，核心是总线协议 `{ say, play[], segue, reason }` 和三层 TTS 降级（WS → HTTP → 浏览器原生）。

## 接入目标

- 将 hanako-audio-player 升级为智能音频播放器
- 实现场景化节目编排
- 集成 TTS 语音串场
- 参考 Claduio 的极光频谱可视化

## 技术方案

- **总线协议**：定义播放器内部事件契约
- **TTS 集成**：复用 hanako-audio-player_tts（CosyVoice 本地克隆）
- **节目编排**：根据时段/场景/情绪自动建播放列表
- **网易云适配**：参考 Claduio weapi 实现（可选）
- **可视化**：Three.js 极光 + 频谱驱动（参考 Claduio 视觉）

## 实施步骤

1. 拆解 Claduio 总线协议，设计播放器事件系统
2. 扩展播放列表管理（场景化：工作/休闲/深夜）
3. 集成 TTS 生成（已有基础）
4. 实现节目编排逻辑（时段/情绪感知）
5. 参考 Claduio 实现极光频谱可视化
6. 网易云音乐适配（Cookie + API）
7. 场景感知（结合 Hanako 当前状态）

## 与现有系统关系

- hanako-audio-player 已有播放列表和 TTS 能力
- Claduio 提供的是"编排层"和"视觉层"
- 可独立演化，不依赖其他插件

## 预期效果

- 播放器从"手动点歌"升级为"场景自动编排"
- 视觉体验大幅提升
- 语音串场增强沉浸感

## 风险与应对

- **风险**：网易云 weapi 加密协议不稳定
- **应对**：优先实现本地音频 + TTS，网易云作为可选扩展
- **风险**：Three.js 可视化性能开销
- **应对**：提供性能模式切换，低配设备降级为 CSS 动画

## 优先级建议

P1-P2 — 体验升级明显，但非核心功能
